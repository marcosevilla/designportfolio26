The Editorial Grid, and how to ask for it

Every page sits on one shared 12-column canvas. Content is placed, not wrapped — you (or Claude) put each block on specific columns at each breakpoint. This doc is the shared vocabulary: say the words in the left columns and the layout on the right happens. Source of truth in code: site/lib/layout-presets.ts.

↓ Page two — the capture list (prototypes → case studies)
01

The canvas

12 equal columns, 24px gaps, max width 1128px (each column ≈ 72px at full size). The canvas is fluid — columns shrink with the window, the composition holds. On case studies the canvas sits to the right of the fixed table of contents.

canvas max 1128px columns 12 × ~72px gap 24px spec "1-6" = columns 1 through 6, inclusive
02

The three bands

Layouts are authored per band. You mostly design the desktop composition; the other two collapse sensibly on their own unless you say otherwise.

< 768px

phone

Everything stacks full-width, in source order. You rarely need to author this band.

base="…"
768 – 1199px

tablet

Mostly still stacked, but with room to breathe — prose narrows, pairs like duo hold side-by-side.

md="…"
≥ 1200px

desktop

The authored composition. Multi-column placements only switch on where they have real room — no awkward squeezed middle state.

lg="…"
Inheritance: an unset band falls back to the one below it — lg → md → base → full width. So <Col lg="1-6"> means "columns 1–6 on desktop, full width everywhere else."
03

The presets — say the name, get the layout

Named compositions. In conversation just use the name: “make the Impact section media-right”, “put the intro on intro-rail.” Orange is the first block inside the section, blue is the second.

04

How to prompt me

Phrases that map one-to-one onto layout changes. Mix freely — everything composes.

Say thisWhat happens
“Make the Solution section prose-wide” That section’s text column grows from 6 to 8 columns on desktop.
“Put the dashboard shot media-right next to the Problem text” Text moves to columns 1–5, image spans 6–12 beside it.
“Full-bleed the guest video” / “make it media-full” The media spans all 12 columns of the canvas.
“Run those two screenshots as a duo” Side-by-side pair, 6 columns each. Holds on tablet, stacks on phone.
“Put the quote at 3-10” (any explicit span works) One-off placement: <Col lg="3-10"> — start column 3, end column 10.
“On tablet make it 2-11 instead” Adds an md spec — same grammar per band.
“Feature the first work cell” / “make the KB cell featured” Home work grid: featured cells span the full canvas, the rest run 2-up.
“Add a meta rail with Year / Role / Scope” The section becomes intro-rail with a MetaRail in the blue slot.
“Alternate these blocks left / right” Blocks take 1-6, 7-12, 1-6, … — the 2×2 spread used for Key Decisions.
“Sheet it” / “run a contact sheet on compendium” npm run sheet -- /work/compendium --unlock → one HTML page with the layout at 390 / 768 / 1024 / 1440 for review.
05

The JSX, if you’re in the code

// a preset assigns columns to children in order
<Grid preset="media-right">
  <Col>…text…</Col>      // → lg 1-5
  <Col>…image…</Col>     // → lg 6-12
</Grid>

// explicit placement (wins over presets)
<Grid>
  <Col md="2-11" lg="3-10">…pull quote…</Col>
</Grid>

// every section is its own <Grid> — identical tracks keep
// columns aligned down the whole page

07

Setup & recording spec

One session setup, reused for every capture below. The prototypes are a state-based router — every screen is a ?page= deep link, no clicking through menus to get there.

server

cd ~/Developer/msevilla-canary-prototypes-1 && npm run dev -- -p 3001 → localhost:3001 (the portfolio dev server owns 3000).

clean frame

Append &demo=1 to every URL — hides the Home button + all dev chrome. ⌘⇧H toggles it live mid-recording.

video spec

MP4, autoplay-loop-muted. Record screen-only: no baked margins or browser chrome, composition stable across the whole loop, native aspect (phone flows are 430×932). This is the lesson from fb-mobile.mp4, whose mid-loop composition change clips scenes in the card crop.

still spec

PNG @2×. 16:9 → 2880×1620 · 3:4 → 1200×1600 · card frame 16:10. Full specs: docs/VISUAL-EXPORT-GUIDE.md.

destinations

Videos → site/public/videos/ · in-page stills → site/public/images/<slug>/ · card composites → site/public/images/gallery/<slug>/ · raw staging → docs/fb-gallery/.

variants

&variant=, &tab=, &flow= lock a specific state for repeatable takes — capture variant comparisons as separate takes, not by clicking the switcher on camera (unless the switch is the story).
